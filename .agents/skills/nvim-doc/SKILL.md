# Neovim 功能文档

为 Neovim 配置中的插件或功能编写中文 Markdown 操作文档。

## 文档位置

所有文档存放在项目根目录的 `docs/` 目录下，文件名使用小写 kebab-case，例如：
- `docs/neo-tree.md`
- `docs/telescope.md`
- `docs/lsp.md`

## 文档结构

生成的文档必须严格按以下结构组织：

```
# [功能/插件名称]（中文简述）

一句话说明该功能/插件的用途。

## 快捷键

[一段简短的说明，介绍该功能/插件的快捷键使用方式和规律，帮助读者快速建立认知。
例如："所有 Telescope 搜索快捷键使用 `<Space>f` 前缀。" 或
"Neo-tree 通过 `<Space>e` 打开，内部使用 j/k/h/l 导航。"
如果快捷键没有明显的前缀规律，则简要说明核心入口快捷键即可。]

### [按用途分类的标题]

| 快捷键 | 模式 | 说明 |
|--------|------|------|
| `<C-k>` | insert | 上一条结果 |

## 常用命令

**必须包含插件原生提供的常用命令和操作方法**，而不仅仅是在 lua 配置中自定义的内容。

这是文档的核心价值所在——用户需要知道插件能做什么，而不仅仅是当前配置了什么。

### 内容来源

1. **从代码提取**：读取 `lua/plugins/<plugin>.lua` 中的自定义命令和快捷键
2. **查阅插件文档**：通过 WebFetch 或 WebSearch 查阅插件的官方文档/README，获取常用命令列表
3. **过滤原则**：只收录**常用**的命令和操作，不常用的不要加入

### 内容格式

每个命令用一行简短说明，必要时附上示例：

\```
| 命令 | 说明 |
|------|------|
| `:MasonInstall <package>` | 安装指定包 |
| `:MasonUninstall <package>` | 卸载指定包 |
\```

对于操作类插件（如 neo-tree、telescope），还需说明窗口内的常用操作：

\```
### 文件树内操作

| 按键 | 说明 |
|------|------|
| `a` | 新建文件/文件夹 |
| `d` | 删除文件/文件夹 |
| `r` | 重命名文件/文件夹 |
\```

**注意**：窗口内的原生按键操作如果没有被自定义覆盖，也应列出，这些是用户日常使用最需要知道的信息。

如何设置和自定义该功能，包含可直接复制的代码片段。

### [配置项标题]

说明配置项的作用。

vim.keymap.set('n', '<Space>ff', builtin.find_files, { desc = '查找文件' })
\```

## 特性

- 用 bullet 列表列出该功能的关键特性

## 快捷键提取

**必须自动从代码中提取快捷键**，不要凭记忆或猜测。

### 提取步骤

1. **全局快捷键**：读取 `lua/config/keymaps.lua`
2. **插件快捷键**：读取对应的 `lua/plugins/<plugin>.lua`
3. 从 `vim.keymap.set()` 调用中提取：
   - 第一个参数 → 模式（`n` = normal, `i` = insert, `v` = visual, `t` = terminal）
   - 第二个参数 → 快捷键
   - `desc` 字段 → 说明文字
4. 从插件 spec 的 `keys = {}` 字段提取快捷键
5. 提取完成后，与现有 `README.md` 中对应部分交叉验证，确保无遗漏

### 模式显示规则

| 代码中的值 | 文档中显示 |
|------------|-----------|
| `'n'` | normal |
| `'i'` | insert |
| `'v'` | visual |
| `'t'` | terminal |
| `'x'` | visual block |
| `{'n', 'v'}` | normal / visual |
| 模式为空或全局 | 省略模式列 |

## 快捷键分组规则

按用途分类组织快捷键表，而非按模式分类：

- **不好的分组**："Normal 模式快捷键"、"Insert 模式快捷键"
- **好的分组**："文件与文本搜索"、"代码操作"、"诊断导航"、"窗口内导航"

当插件有多种使用场景时，用 `###` 子标题分开。模式信息放在表格的"模式"列中。

## 配置说明要求

- 给出可直接复制的 Lua 代码片段，不要用纯文字描述
- 代码片段应展示最小可用的配置示例
- 如果配置较复杂，分多个子标题逐步说明
- 标注配置项所在的文件路径，方便读者查找完整配置

## 语言和风格

- 全部使用中文
- 快捷键用内联代码格式（反引号）
- 表格是核心呈现形式，快捷键必须用表格展示
- 每个分类标题下先放快捷键表格，再放补充说明
- 保持简洁，避免冗余描述

## 特殊按键表示

| 实际按键 | 文档表示 |
|---------|---------|
| Ctrl | `<C-x>` |
| Alt | `<A-x>` 或 `Alt-x` |
| Leader | `<Space>`（本项目 Leader 键为 Space） |
| Escape | `Esc` |
| Enter/Return | `<CR>` |
| Tab | `<Tab>` |
| Backspace | `<BS>` |

## 生成后检查

生成文档后必须检查：

1. **快捷键完整性**：与源码中的 `vim.keymap.set` 调用逐一对比，确保无遗漏
2. **快捷键准确性**：每个快捷键的按键组合和模式与代码一致
3. **与 README 一致**：文档中的信息不与 `README.md` 中的现有描述矛盾
4. **代码片段可运行**：给出的配置代码可以直接复制使用
