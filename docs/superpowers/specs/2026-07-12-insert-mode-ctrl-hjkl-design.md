# 插入模式 Ctrl+hjkl 光标移动

- 日期：2026-07-12
- 状态：已批准，待实现

## 背景

当前 Neovim 配置在插入模式下无法用主键区移动光标，只能退出插入模式或伸手到键盘左下角按方向键，影响编辑流畅度。希望在插入模式下用 `<C-hjkl>` 实现光标移动，让双手不必离开主键区。

## 需求

- `<C-j>` / `<C-k>`：插入模式下上 / 下跨行移动，**保持当前列位置**（跨短行时停在行尾）。
- `<C-h>` / `<C-l>`：插入模式下左 / 右单字符移动。
- `<C-j>` / `<C-k>` 在补全菜单弹出时**始终移动光标**，不切换补全项（补全项仍用 `<Tab>` / `<S-Tab>`）。
- 不影响 normal 模式下已有的 `<C-hjkl>` 窗口焦点切换映射。

## 冲突确认

插入模式下 `<C-hjkl>` 此前均未被显式映射，但存在以下默认语义会被覆盖：

| 按键 | 原默认行为 | 覆盖后影响 | 决策 |
|------|-----------|-----------|------|
| `<C-h>` | Backspace | 失去插入模式退格，物理 `<BS>` 键仍可用 | 接受 |
| `<C-j>` | 插入 `<NL>`（换行） | 失去 `<NL>` 换行（`<CR>` 仍可用） | 接受 |
| `<C-k>` | digraph 输入（`©`、`→` 等） | 放弃 digraph 输入能力 | 接受 |
| `<C-l>` | 无特殊行为 | 无影响 | — |

normal 模式下 `<C-hjkl>` 已映射为窗口焦点切换（`lua/config/keymaps.lua:74-77`），与插入模式互不干扰，不改动。

## 设计

### 方案选择

采用方案 A+B 混合：将 `<C-hjkl>` 映射到 Vim 插入模式的原生移动键 `<Left>` / `<Right>` / `<Up>` / `<Down>`。原生移动键自动处理列位置保持、短行行尾、虚拟列等边界情况，无需手写逻辑。

### 映射定义

在 `lua/config/keymaps.lua` 中追加（紧跟现有 jk 退出插入模式映射之后）：

```lua
-- 插入模式下用 Ctrl+hjkl 移动光标，避免手离开主键区够方向键
-- 覆盖了 <C-h>(Backspace) 和 <C-k>(digraph)，退格用物理 <BS>，digraph 不再可用
map("i", "<C-h>", "<Left>", { desc = "插入模式：左移一字符", silent = true })
map("i", "<C-l>", "<Right>", { desc = "插入模式：右移一字符", silent = true })
map("i", "<C-j>", "<Down>", { desc = "插入模式：下移一行", silent = true })
map("i", "<C-k>", "<Up>", { desc = "插入模式：上移一行", silent = true })
```

### 行为对照

| 按键 | 新行为 | 原默认行为（被覆盖） |
|------|--------|---------------------|
| `<C-h>` | 左移一字符 | Backspace |
| `<C-l>` | 右移一字符 | 无 |
| `<C-j>` | 下移一行，保持列 | 插入 `<NL>` |
| `<C-k>` | 上移一行，保持列 | digraph 输入 |

### 不改动的部分

- `lua/plugins/cmp.lua`：cmp 未映射 `<C-hjkl>`，补全菜单交互无冲突。
- normal 模式的窗口切换映射：保持不变。
- terminal 模式：不添加 `<C-hjkl>` 映射。终端里这些键常被 shell 程序自身使用（行编辑等），覆盖会干扰 shell 操作。
- 不新增插件、不加依赖。

## 验证

按 AGENTS.md 约定，插入模式交互无法在 headless 下可靠测试，需手动验证：

1. 打开文件进入插入模式，逐个测试 `<C-hjkl>` 四个方向移动。
2. 验证跨短行时列位置保持、停在行尾的行为。
3. 触发补全菜单（输入代码），确认 `<C-j>` / `<C-k>` 移动光标而非选择补全项。
4. 语法验证：用 `nvim --headless "+qa!"` 启动完整配置，确认无加载错误（keymaps.lua 依赖 nvim runtime，不能 `-u NONE` 单独加载）。
